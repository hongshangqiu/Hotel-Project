## Taro 跨端开发

### Taro 优势
| 特性 | 说明 |
|------|------|
| 跨平台 | 一套代码，同时编译到 H5、微信小程序 |
| 性能高 | 运行时优化，接近原生体验 |
| 生态好 | 丰富的组件和插件支持 |
| 维护方便 | 统一技术栈，减少多端维护成本 |

### 命令说明
| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动 H5 开发服务器（端口 10086） |
| `npm run dev:weapp` | 启动微信小程序开发模式 |
| `npm run build` | 构建 H5 生产版本 |
| `npm run build:weapp` | 构建微信小程序版本 |

### 组件使用规范
```jsx
// ❌ 错误写法（不应使用原生 HTML 标签）
<div>内容</div>
<span>文本</span>

// ✅ 正确写法（使用 Taro 组件）
import { View, Text, Image } from '@tarojs/components'

<View>内容</View>
<Text>文本</Text>
<Image src="图片地址" />

// 在 H5 模式下会自动转换为 HTML 标签
// 在小程序模式下会自动转换为对应组件
```

### Taro API 使用
```jsx
import Taro from '@tarojs/taro'

// 提示框
Taro.showToast({
  title: '操作成功',
  icon: 'success'
})

// 导航
Taro.navigateTo({
  url: '/pages/detail/index?id=1'
})

// 获取当前页面实例
const currentPage = Taro.getCurrentInstance()
```

## 注意事项

1. **组件使用规范**：必须使用 Taro 组件（View、Text、Image 等），不能使用原生 HTML 标签
2. **路由配置**：所有页面需要在 app.config.js 中注册
3. **样式隔离**：每个页面都有自己的样式文件，使用 SCSS 预处理器
4. **状态管理**：使用 Zustand 进行全局状态管理
5. **API 请求**：使用 Axios 进行 HTTP 请求
6. **日期处理**：使用 Dayjs 进行日期格式化
