

缺陷说明（无后端，无法更改）

### 3.1 商户 ID 生成问题
- **位置**：登录页第 91 行、注册页第 82 行
- **问题**：使用 `Date.now()` 生成 ID
- **影响**：同一账号每次登录得到不同 ID，不符合实际业务逻辑
- **正确做法**：ID 应由后端数据库生成并返回

### 3.2 安全性问题

#### 3.2.1 密码明文存储
- **位置**：
  - `LocalStorage.ts` 第 58 行：`u.password === pwd`
  - 注册页第 92 行：`password: password`
- **问题**：密码以明文形式存储在 LocalStorage
- **真实项目做法**：
  - 密码应哈希存储（如 bcrypt）
  - 前端不存储密码，只传输用于验证

#### 3.2.2 密码明文比较
- **位置**：登录验证逻辑
- **问题**：直接比较 `u.password === pwd`
- **真实项目做法**：后端验证密码哈希，前端不参与密码比对

#### 3.2.3 固定 Token
- **位置**：登录页第 104 行、注册页第 85 行
- **问题**：Token 值为固定或简单拼接的 mock 值
- **真实项目做法**：
  - 后端生成 JWT token
  - 包含用户 ID、角色、过期时间
  - 每次登录生成新 token

### 3.3 逻辑问题

#### 3.3.1 角色由前端判断
- **位置**：登录页第 40-52 行
- **问题**：角色由前端根据账号名判断，而非后端返回
- **真实项目做法**：登录接口返回用户信息（含角色），前端信任后端数据

#### 3.3.2 注册页可选管理员角色
- **位置**：注册页第 173-191 行
- **问题**：普通用户可以自行选择注册为管理员
- **真实项目做法**：管理员账号由后台手动创建，普通用户无法自行注册为管理员

#### 3.3.3 登录成功后无错误处理
- **问题**：登录成功后直接跳转，无 try-catch 错误处理
- **真实项目做法**：网络请求应有完整的错误处理机制
