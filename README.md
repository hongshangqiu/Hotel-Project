# 易宿酒店预订平台 (Easy Stay Hotel Booking Platform)

## 项目介绍

易宿酒店预订平台是一个面向现代旅游出行场景的综合服务体系，旨在为酒店商家与终端消费者之间搭建高效、便捷的信息交互桥梁。

本项目分为两部分，分别是商户端的管理酒店信息平台和用户端的酒店预定流程页面。通过差异化的产品设计满足不同用户群体的核心诉求。

- **商户端平台**：专注于酒店信息管理与运营协作，为酒店商家提供专业化的信息录入、审核流程及发布管理能力。
- **用户端平台**：聚焦于用户体验优化，通过简洁直观的交互流程帮助用户快速找到符合需求的优质酒店。

> 本项目为 **第五期前端训练营** 大作业。

## 技术栈

本项目采用 **Taro** 框架，结合 **React** 与 **TypeScript**，实现“一次开发，多端运行”。

### 核心架构

- **基础框架**: [Taro 3](https://taro-docs.jd.com/taro/docs/) (React + Webpack 5)
- **开发语言**: TypeScript (TS) - 统一定义酒店、用户、订单的接口规范（Interfaces）
- **状态管理**: [Zustand](https://github.com/pmndrs/zustand) - 核心枢纽，存储登录状态、选中的酒店、搜索条件，双端共用
- **UI 组件库**: [NutUI-React](https://nutui.jd.com/taro/react/1x/#/zh-CN) - 京东出品，完美适配 Taro，且在 PC 端显示效果比传统小程序组件更优
- **样式处理**: SCSS + CSS Modules - 利用变量控制移动端和 PC 端的视觉差异
- **HTTP 客户端**: Axios - 封装请求拦截与响应拦截

### 目录结构

```text
src/
├── shared/               # 共享逻辑层（核心资产）
│   ├── types/            # TS 类型定义 (IHotel, IUser, IOrder)
│   ├── store/            # Zustand 状态池 (useStore)
│   ├── services/         # Axios 请求封装 (API 接口)
│   └── utils/            # 通用工具函数 (日期格式化, 权限判断)
├── pages/                # 视图层（分端适配）
│   ├── user/             # 用户端 (移动端布局)
│   │   ├── index/        # 酒店查询页 (首页)
│   │   ├── list/         # 酒店列表页
│   │   └── detail/       # 酒店详情页
│   └── admin/            # 管理端 (PC端布局)
│       ├── login/        # 登录/注册
│       ├── manage/       # 信息录入/编辑
│       └── audit/        # 审核/发布列表
├── app.config.ts         # 路由总配置
├── app.scss              # 全局样式
└── app.tsx               # 入口文件
```

## 快速启动

### 环境要求

- Node.js (20.18)
- npm 

### 安装依赖

在项目根目录下执行：

```bash
npm install
```

### 启动开发环境

- **H5 端 (推荐用于调试 PC/移动端响应式)**:
  ```bash
  npm run dev:h5
  ```
- **微信小程序**:
  ```bash
  npm run dev:weapp
  ```

打开浏览器访问显示的地址
